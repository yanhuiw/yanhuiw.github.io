Java 常用类

装箱 & 拆箱

箱装：编译器支持的包装。包装类型的比较必须使用equals()。包装类型提供了大量实用方法。

Java 核心库提供的包装类型可以把基本类型包装成类，也可以把类拆解成基本类型。两者都只发生在编译阶段，可以少写代码，但会影响代码的执行效率。

  • 自动装箱：直接把基本类型变成引用类型的赋值写法。

  • 自动拆箱：直接把引用类型变成基本类型的赋值写法。可能会出现null无法拆箱的异常。

Number

  • byte，int，short，long，double，float分别对应Byte，Integer，Short，Long，Double，Float包装类。这些包装类都是抽象类Number的子类。

  • 支持自动装箱和自动拆箱。

  • 创建引用对象：
    • 创建新的实例：Integer n=new Integer(100);
    • 静态工厂方法：Integer n=Integer.valueOf(100); 优先选用。具体实现可以跟随版本进行优化。尽可能地返回缓存的实例以节省内存。
  • 使用equals进行等值比较
  • 常用方法

BigInteger

  • 内部使用一个int[]数组模拟一个非常大的整数。没有范围限制，但速度较慢。

  • BigInteger是不变类，继承自Number。

  • 对BigInteger做运算的时候，只能使用实例方法。

  • 可以把BigInteger转换成byte，short，int，long等基本类型，可以判断是否超出转换后类型的范围。
  • 可以把BigInteger转换成float，double等基本类型，如果超出范围返回inf。

BigDecimal

  • 表示一个任意大小且精度完全准确的浮点数。

  • 小数位数：获得小数位数 scale()，设置小数位数 setScale()。

  • 去掉末尾的0：stripTrailingZeros()。

  • 等值比较：compareTo()。不能使用equals，因为它要求两个值相等，且小数位数相等。

Character

  • 基本类型 char 的包装类 Character。

  • 支持自动装箱和自动拆箱。

  • 常用方法

String

字符串特性

String 是一个引用类型，本身也是一个类。

String 内部使用一个 char[] 数组表示。

字符串不可变：内部使用 private final char[] 存储字符串。

字符串比较：必须使用 equals()。因为字符串的不可变性，”hi”和”HI”.toLowerCase()不相等。

常用操作

比较，获得长度，提取子串，查找，替换，大小写转换，连接字符串

字符编码

  • ASCII码：1个字节，包含英文字母，数字和常用符号。
  • GB2312：2个字节，第1个字节的最高位始终为1，用于区分ASCII 码。包含中文。
  • Unicode：2个字节，包含世界上的常用语言。
  • UTF-8：1-4 字节变长编码，依靠高位字节确定一个字符的长度，某处字符出错不影响后序字符，容错能力强，常用作传输编码。

StringBuffer & StringBuilder

字符串拼接的时候，每次都会创建新的字符串对象，删除旧的字符串，导致大部分字符串都是临时对象，占用时间和空间。

StringBuilder 和 StringBuffer 都是一个可变对象，可以预分配缓冲区，拼接字符串时，不会创建新的临时对象，提高了效率。

StringBuilder 和 StringBuffer 可以进行链式操作，每一次操作都返回实例本身。

StringBuffer 是线程安全的。在应用程序有此要求的情况下必须使用 StringBuffer。StringBuilder 不是线程安全的，但效率更高。

JavaBean

一种符合命名规范的类，通过getter和setter来定义属性。

  • private实例字段：private Type value;
  • public方法读写实例字段，且读写方法符合以下命名规范：
    • public Type getValue(){}; public boolean isValue(){};
    • public void setValue(Type value){};

属性

  • 通常把一组对应的读方法和写方法称为属性。属性只需要定义读和写方法，不一定需要对应的字段。

    • 只读属性：只有读方法
    • 只写属性：只有写方法

  • 可以利用IDE快速生成getter和setter。

  • 可以使用Introspector.getBeanInfo()获取属性列表。

enum

定义常量，并让编译器能够自动检查某个值在枚举的集合内。

编译器检查

  • enum常量自带类型信息，编译器会检查互相比较或者赋值的类型是否相同，如果不同则编译错误。

  • 检查是否引用到非枚举的值，如果有则编译错误。

enum比较

enum类型的每个常量在JVM中只有一个唯一实例，可以直接用==比较。

enum类型特点

  • 定义的enum类型总是继承自java.lang.Enum，且无法被继承。
  • 只能定义处enum的实例，无法通过new操作符创建enum实例。
  • 定义的每个实例都是引用类型的唯一实例。
  • 可以将enum类型用于switch语句。
  • 可以定义枚举类型的字段，输出更多信息。
