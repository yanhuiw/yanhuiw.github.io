Java 面向对象编程

Posted on 2019-10-29 In Java Views:

Java 面向对象编程

对象和类

对象和类的概念

  • 对象：类的一个实例，有状态和行为。

  • 类：对象的一个模板，描述一类对象的状态和行为。

举例：

  • 类：person。

  • 对象：Bob，Alice 等具体的某个人。

一个类可以包含：

  • 字段（field）/变量：描述某一类对象的状态。
    • 局部变量：在方法，构造方法或者语句块中定义的变量，方法结束后自动销毁。
    • 成员变量：在类中，方法体之外的变量。属于对象，在创建对象的时候实例化。可以被类中的方法，构造方法或者语句块访问。
    • 类变量：在类中，方法体之外的 static 类型变量，属于类。所有同一类的实例共享一个类的静态变量。
  • 方法（function）：描述某一类对象的行为。
  • 构造方法：类的初始化方法。

方法

  • 给外部代码提供一个安全访问类内字段的途径。

  • 在方法内部可以进行参数安全检查。

构造方法

  • 构造方法用于在创建实例的时候，对实例进行初始化。
  • 每个类都有至少一个构造方法。如果有多个构造方法，则 Java 编译器通过参数数量，位置，类型自动区分。
  • 如果没有显式定义构造方法，Java 编译器会为该类提供一个默认的构造方法。如果显式定义了构造方法，编译器就不再自动创建默认构造方法。
  • 构造方法必须与类同名。
  • Java 创建对象实例时，先进行字段初始化，再执行构造方法进行初始化。

静态字段和静态方法

  • 静态字段
    • 属于类。同一个类的所有实例共享一个静态字段。
    • 推荐使用类名.静态字段来访问静态字段。实例对象访问静态字段时，编译器自动转换成类名方式访问。
  • 静态方法
    • 属于类。无法访问this变量，无法访问实例字段，只能访问静态字段。
    • 推荐使用类名.静态方法来调用静态方法。实例对象调用静态方法时，编译器自动转换成类名方式调用。

封装

  • 隐藏对象的属性和实现细节，仅对外公开接口。控制对象的读写访问级别。

  • 目的：

    • 增加安全性：使用者不了解对象的属性和实现细节，只能通过开放的接口进行对象的操作。类内可以对外部操作进行安全检查。
    • 简化编程：使用者无需了解对象的属性和实现细节，只要通过开放的接口进行对象的操作即可实现目标。

继承

  • 目的：使得子类具有父类的属性和方法，实现代码复用。

  • 术语：

    • 被继承者：超类，父类，基类。
    • 继承者：扩展类，子类，派生类。

  • 关系：子类 is a 父类。父类更通用，子类更具体。

  • 例子：生物作为父类，动物和植物作为子类。动物/植物都是一种生物，都包含生物的基本特征，符合 is-a 关系。同时，动物和植物都有各自独特的属性和行为。

  • 关键字

    • extends：is-a 继承。只能继承一个父类。
    • implements：has 继承。可以继承多个接口。
    • final：
      • 修饰类：表示该类不能被继承。
      • 修饰方法：表示该方法不能被重写。
      • 修饰字段：表示该字段初始化后就不能被修改。

  • 继承类型

    • 单继承：一个子类只能继承一个父类。
    • 多重继承：多个单继承串联，A类继承B类，B类继承C类。所有类最终的根类是 Object。
    • 多继承：Java 不允许一个子类继承多个父类。但允许一个子类继承多个接口。

  • 访问权限：子类可以访问父类由 public 和 protected 修饰的字段和方法。

  • 构造方法：子类不继承父类的构造方法或者构造函数，但可以调用。

    • 父类构造方法无参数：系统自动调用父类的无参数构造方法。
    • 父类构造方法有参数：子类的构造方法可以通过 super() 显式调用父类的构造方法。

  • 转型

    • 向上转型（upcasting）：把子类类型安全地变为父类类型的赋值。

    • 向下转型（downcasting）：把父类类型强制转型为子类类型。

      向下转型可能会失败。向下转型成功要求：转型前的实例指向的类型是转型后的类型及其子类。

      可以先使用 instanceof 判断一个实例是不是某种类型，再进行向下转型。

多态

重载与重写

  • 重载（Overload）：子类定义了一个与父类方法名称相同，但签名不同的方法。

    重载的规则：

    • 被重载的方法必须改变参数列表，可以改变返回类型，可以改变访问修饰符。
    • 被重载的方法可以声明新的或更广的检查异常。
    • 方法能够在同一个类中或者在一个子类中被重载。

  • 重写（Override）：子类定义了一个与父类方法签名完全相同的方法。外壳不变，核心重写。

    重写的规则：

    • 子类方法的访问权限必须大于等于父类的访问权限。（public>protected>private）
    • 父类的成员方法只能被它的子类重写。
    • 只有声明为 public 和 protected 的非 final 方法才可以被重写。
    • 构造方法不能被重写。
    • 重写的方法能够抛出任何非强制异常，但不能抛出新的或者更广的强制性异常。

多态

加@Override可以让编译器帮助检查是否进行了正确的重写。非必需。

针对某个类型的方法调用，是基于运行时的实际类型的动态调用，而非变量的声明类型。

允许添加更多类型的子类实现功能扩展，而不需要修改基于父类的代码。

例：假设各科成绩总分都是100，不同科目加权值不同，计算总成绩。

重写 Object 方法

所有的类最终都继承自 Object。Object 类中有几个重要的方法常被重写：

  • toString()

  • equals()

  • hashCode()

非多态的情况

  • super：显式调用父类被重写的方法。在编译期确定。

  • final：使用final修饰的方法无法被重写，不存在多态。

抽象类

如果父类的方法本身不需要实现任何功能，仅仅是为了定义方法签名，目的是让子类重写，那么可以把父类的方法声明为抽象方法，使用abstract修饰。

抽象类是包含抽象方法的类，使用abstract修饰。抽象类无法被实例化，只能被继承。

如果子类不实现抽象方法，则该子类仍然是一个抽象类。

面向抽象编程：

  • 通过抽象类去引用具体的子类实例。

  • 上层代码（抽象类）定义规范，具体的业务逻辑由不同的子类实现。

  • 调用者只关心抽象方法的定义，不关心子类的方法具体是怎么实现的。

接口

接口（interface）：没有字段，所有方法都是抽象方法的抽象类。

接口的继承：

  • 类实现接口：implements。一个类可以实现多个接口。

  • 接口扩展接口：extends。

default方法：放需要给接口新增一个方法时，会设计到修改全部子类。如果新增default方法，那么子类就不需要全部修改，只需要在用到的地方重写方法。

包

包（package）。

包的作用

  • 把功能相似或相关的类或接口组织在同一个包中，方便类的查找和使用。

  • 避免类名冲突。同一个包中的类名必须不同，不同包中的类名可以相同。编译器最终编译出的.class文件使用完整类名。如果使用了简单类名，按照当前package->import 的package->java.lang的顺序查找类。

  • 限定访问权限。拥有包访问权限的类才能访问包中的类。

包的结构

包采用树形目录的存储方式。Java 文件对应的目录层次和包的层次一致。例：

创建一个 student.work.sample.java 的包，那么该包有三层文件结构，student 文件夹中存放 work 文件夹，work 文件夹中存放 sample.java 文件。

没有包名的类使用默认包，容易引起类名冲突。

包没有父子关系。 例：java.util 和 java.util.zip 是不同的包，两者没有任何继承关系。

包的作用域

不用public，protected，private修饰的字段和方法，可以被包内的类访问。

包的引用

import

  • 引入包中的类：import java.util.Arrays;

  • 引入整个包：import java.util.*;

常用的包

  • java.lang：存放Java 核心类，编译器自动导入。
  • java.util
  • java.math
  • java.text
