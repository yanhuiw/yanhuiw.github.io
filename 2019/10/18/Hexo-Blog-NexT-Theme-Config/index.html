前言

Github上有许多精致且实用的主题模板，详情可参考 https://www.zhihu.com/question/39388850

主题美化

打开 themes\next文件夹中的_config.yml文件进行设置。若选择其他主题，设置相应主题的_config.yml文件即可。

外观 - scheme

  • Muse - 默认 Scheme，NexT 最初的版本，黑白主调，大量留白
  • Mist - Muse 的紧凑版本，整洁有序的单栏外观
  • Pisces - 双栏 Scheme，小家碧玉似的清新
  • Gemini - 紧凑集中版双栏 Scheme，本网站采用的主题

将选中的主题前的#去掉，其他加#。

语言 - language

编辑站点配置文件下的language，修改所有主题的语言。

编辑某一主题配置文件下的language，修改当前主题的语言。

NexT 语言支持：

菜单 - menu

在menu字段下选择启用的菜单，也可以新增自定义的菜单。

常用的菜单如下：

键值 设定值 显示文本（简体中文）
home home: / 主页
archives archives: /archives 归档页
categories categories: /categories 分类页
tags tags: /tags 标签页
schedule schedule: /schedule 日程表
about about: /about 关于
sitemap sitemap: /sitemap 站点地图
commonweal commonweal: /404.html 公益404

在主题目录的language文件夹中，修改相应的语言包的菜单显示文字。

侧栏 - sidebar

修改sidebar字段下的position的值，设置侧栏的显示位置。目前仅 Pisces scheme 支持position配置。

position支持以下的选项：

  • left - 靠左放置
  • right - 靠右放置

修改sidebar字段下的display的值，设置侧栏的显示时机。

display支持以下的选项：

  • post - 默认行为，在文章页面（拥有目录列表）时显示
  • always - 在所有页面中都显示
  • hide - 在所有页面中都隐藏（可以手动展开）
  • remove - 完全移除

头像 - avatar

将头像图片放在主题目录的source/images文件夹下，图片名称设为avatar。

如果_config.yml没有avatar字段，则新增字段avatar。并设置 url 参数。
